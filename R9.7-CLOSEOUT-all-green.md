## §0 — Reviewer error acknowledged

My previous probes (R9.7-A, R9.7-B) reported `/api/health/detail` and `/api/_routes` returning 200 unauth. **That was wrong.** Default `fetch()` in a browser tab sends `credentials:'same-origin'`, which auto-attaches the SPA's session cookie. I was measuring authenticated behavior while believing I was anonymous.

Re-probed with `credentials:'omit'` to match a true unauthenticated client (matches `curl`'s zero cookie jar):

```
GET  /api/health           → 200  {"ok":true,"ts":...,"state":"ready"}
GET  /api/health/detail    → 401  {"error":"Authentication required"}
GET  /api/_routes          → 401  {"error":"Authentication required"}
POST /api/internal/audit   → 403  {"ok":false}
POST /api/auth/cli-mint    → 403  {"ok":false}
GET  /.well-known/security.txt → 200
```

Matches the agent's curl table exactly. Container is correct. Code is correct. R9.7 closes.
