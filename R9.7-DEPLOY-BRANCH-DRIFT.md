## §2 — Root cause

The production deploy on the server (whatever script / compose / watchtower / git-pull cron is doing it) is configured to track `main`. Every round from R1 through R7 must have been merged into `main` at some point, which is why those fixes are live. R8 onwards stopped at `dev`.

This is a 1-line config issue, not a code issue. Either:

**(A)** The deploy script does `git pull origin main` → switch it to `git pull origin dev` (if dev IS the prod branch by design).

**(B)** The deploy script tracks `main` (the correct prod pattern) → merge `dev` into `main` so the fixes propagate.

(B) is the standard pattern and what every other round assumed.
