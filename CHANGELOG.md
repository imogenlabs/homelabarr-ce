### 🔒 Security
- **Fix SQLCipher encryption-at-rest initialisation** (Epic [HLCE-209](https://mjashley.atlassian.net/browse/HLCE-209), HLCE-256): `server/db.js` applied `journal_mode = WAL` **before** the SQLCipher `cipher`/`key` pragmas, which initialised a fresh database file as plaintext and then made the key application fail with `file is not a database`. As a result, encryption-at-rest could never initialise — setting `SQLCIPHER_KEY` (≥32 chars) on first boot would abort the backend at startup. The key pragmas now run immediately after opening the connection, before `journal_mode`. A fresh encrypted DB round-trips and writes a ciphertext header, an existing encrypted DB reopens with the correct key, and a wrong key is rejected. (Found and pinned by HLCE-221's persistence tests; the pinned regression test in `server/db.test.js` is now flipped to assert the correct behaviour.)
