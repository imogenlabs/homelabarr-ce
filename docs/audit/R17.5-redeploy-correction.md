## §1 — Root cause hypotheses

Three non-exclusive candidates, in decreasing order of likelihood for a "deploy live on server" workflow:

1. **Frontend image rebuild didn't happen.** R17 changed `index.html` (Vite entry point). Vite builds at image build time and bakes the bundle in. If `docker compose pull && docker compose up -d` ran without rebuilding the frontend image — or pulled an older tag — the new HTML never gets into the served artifact. Likelihood: HIGH. The `last-modified` of 2 hours pre-merge points directly at this.

2. **nginx config template was updated in the repo but not re-rendered on the host.** If the nginx container uses `envsubst` or similar to render the template at startup, and the container wasn't restarted (or its config-render init didn't re-run), the old config is still active. Likelihood: HIGH for the location-directive failures specifically.

3. **GHCR image tag mismatch.** R17 commit on main produces a new image tag (e.g. `:main` or `:sha-139ce795`), but the deployment pulled `:latest` or a pinned older tag. Likelihood: MEDIUM.

The pattern is identical to what surfaced in **R9.7** — source ahead of running container — which was eventually traced to the deploy step not picking up the most recent build.

## §2 — Action: required steps on the deploy host

These are concrete steps for the deploy host. Agent can run them if it has SSH/host access; otherwise this is the runbook the owner follows.


## §7 — Lessons added to the loop

1. **Source verification ≠ live verification.** Going forward every round that touches anything served to the public (nginx config, frontend bundle, headers, response bodies) must include a post-deploy live probe **with the cache-busted, credentials:'omit' pattern**, not just a source diff against the repo.

2. **Deploy currency probe.** Add to every round's §0 verification: `curl -sI <live> | grep last-modified` compared against `gh api /repos/{owner}/{repo}/commits/main --jq .commit.committer.date`. If live is older than main, flag as a deploy-drift finding before the round's substantive verification even starts.

3. **Surface checks belong on the live origin, not the source.** Specifically for R17-class rounds: when the spec is "what does an outsider see," the verification has to *be* an outsider — meaning a probe of the deployed artifact, not a grep of the repo.

These three lessons get folded into the standing loop template (mentally; no doc change needed — they apply to future rounds, not retroactively to R1–R17).
