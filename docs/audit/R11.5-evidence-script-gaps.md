## §1 — Findings

### L-1 — Add audit-chain integrity evidence

**WRONG:** `compliance/collect-evidence.sh` does not emit `evidence/R6-audit-chain.txt`. The R6 audit-log hash-chain claim (referenced by ASVS V7.1.1, V7.3.1, NIST DE.AE-03) currently has no auto-generated proof. A future run of R11 §4 verify step `grep -q '"bad":0' compliance/evidence/R6-audit-chain.txt` would fail.

**RIGHT:** Append to `compliance/collect-evidence.sh`:

```sh
# --- R6 audit log chain integrity ---
{
  echo "# R6 audit chain integrity (last 50 events)"
  echo "# timestamp=$TS git=$HEAD"
  echo
  docker exec "$BACKEND" sh -c "tail -n 50 /app/data/audit.log" | node -e '
    let buf = "";
    process.stdin.on("data", d => buf += d);
    process.stdin.on("end", () => {
      const lines = buf.trim().split("\n").filter(Boolean).map(JSON.parse);
      let ok = 0, bad = 0;
      for (let i = 1; i < lines.length; i++) {
        if (lines[i].hash_prev === lines[i-1].hash_curr) ok++;
        else bad++;
      }
      console.log(JSON.stringify({ ok, bad, total: lines.length }));
    });
  '
} > "$OUT/R6-audit-chain.txt"
```

Expected output:

```
# R6 audit chain integrity (last 50 events)
# timestamp=2026-05-23T01:30:00Z git=4200732838

{"ok":49,"bad":0,"total":50}
```

Any `bad > 0` is a tamper alert — the next compliance-evidence workflow run will fail loudly.

### L-2 — Add cosign verify evidence

**WRONG:** `collect-evidence.sh` does not emit `evidence/R5-cosign.txt`. R5 supply-chain claims (CIS 4.2, CIS 4.5, ASVS V14.1, NIST PR.PS-01, GV.SC-01) have no auto-generated proof.

**RIGHT:** Append to `compliance/collect-evidence.sh`:

```sh
# --- R5 cosign verify on latest image ---
{
  echo "# R5 cosign verify on latest image"
  echo "# timestamp=$TS git=$HEAD"
  echo
  IMG=$(docker compose config --no-interpolate 2>/dev/null | yq -r '.services.backend.image' 2>/dev/null \
        || docker inspect "$BACKEND" | jq -r '.[0].Config.Image')
  echo "Image: $IMG"
  echo
  if command -v cosign >/dev/null 2>&1; then
    cosign verify \
      --certificate-identity-regexp 'smashingtags' \
      --certificate-oidc-issuer https://token.actions.githubusercontent.com \
      "$IMG" 2>&1 | head -40
  else
    echo "cosign not installed on this runner — install in workflow before collecting"
  fi
} > "$OUT/R5-cosign.txt"
```

And update `.github/workflows/compliance-evidence.yml` to install cosign before running the script:

```yaml
      - name: Install cosign
        uses: sigstore/cosign-installer@<PIN-SHA>
        with:
          cosign-release: 'v2.4.0'
```
